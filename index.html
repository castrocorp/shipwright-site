Features

Everything you need.
Nothing you don't.

01

TDD-First

Unit tests before implementation, every time. Up to 30 Red-Green-Refactor iterations until all criteria pass.

02

Code Review Agent

Isolated agent that never sees implementation decisions. Reviews 6 dimensions: standards, bugs, security, history, PRs, smells.

03

JIRA Native

Automatic status transitions, acceptance criteria analysis, ticket type detection for branch naming, and linked ticket review.

04

Slack Tracking

One thread per task. Every pipeline step posts updates. Full audit trail from ticket to merged PR.

05

Lessons Learned

Record incidents with /record-lesson. Before every implementation, /check-lessons surfaces relevant past mistakes.

06

Stack Adapters

Language-specific code quality rules injected into TDD and review. Ships with Kotlin/Spring, extensible via template.

Why Shipwright

Stop orchestrating.
Start shipping.

Manual workflow

  • — Read JIRA ticket, copy AC manually
  • — Create branch, remember naming convention
  • — Write tests, hope you covered edge cases
  • — Implement, forget to check past incidents
  • — Self-review with your own biases
  • — Create PR, update JIRA, post to Slack
  • — One ticket at a time

Shipwright

  • › AC extracted and cross-referenced automatically
  • › Branch type detected from JIRA ticket type
  • › TDD loop with stack-specific quality rules
  • › Past incidents reviewed before every implementation
  • › Independent 6-dimension agent review
  • › PR, JIRA transition, and Slack — all automatic
  • › 2-5 tickets in parallel via worktrees
Configuration

Five layers.
Zero duplication.

Engine rules, universal rules, stack adapters, and project config cascade cleanly. Personal overrides never conflict with shared workflow.

Engine ~/.claude/CLAUDE.md All projects
Rules ~/.claude/rules/*.md Auto-loaded
Stack ~/.claude/stacks/{stack}.md Per language
Personal ~/.claude/CLAUDE.local.md Your overrides
Project .claude/project.md Per repo
